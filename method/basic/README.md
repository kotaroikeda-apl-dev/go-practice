# Go Methods の基本

このディレクトリでは、Go のメソッドに関する基本的な概念を順を追って学びます。

## 学習順序

### 1. [struct - 構造体のメソッド](./struct/README.md)

構造体に対してメソッドを定義する基本的な方法を学びます。

- 値レシーバーを使ったメソッドの定義
- メソッドの呼び出し方
- String()メソッドの実装

### 2. [function - ポインタと関数](./function/README.md)

メソッドではなく通常の関数として、ポインタと値の違いを学びます。

- 値で受け取る関数とポインタで受け取る関数の違い
- メソッドとの違い

### 3. [nonstruct - 構造体以外の型のメソッド](./nonstruct/README.md)

構造体以外の型（エイリアス型）にメソッドを定義する方法を学びます。

- エイリアス型の定義
- 非構造体型へのメソッド定義

### 4. [pointer - ポインタレシーバー](./pointer/README.md)

ポインタレシーバーを使ったメソッドの定義方法を学びます。

- 値レシーバーとポインタレシーバーの違い
- ポインタレシーバーで状態を更新する方法

### 5. [indirection - メソッドとポインタの間接参照（ポインタレシーバー編）](./indirection/README.md)

ポインタ引数を持つ関数とポインタレシーバーを持つメソッドの呼び出し方の違いを学びます。

- 値から呼び出す場合とポインタから呼び出す場合
- 関数はポインタを明示的に渡す必要がある
- メソッドは値でもポインタでも呼び出せる（Go が自動変換）

### 6. [indirection-value - メソッドとポインタの間接参照（値レシーバー編）](./indirection-value/README.md)

値引数を持つ関数と値レシーバーを持つメソッドの呼び出し方の違いを学びます。

- 値から呼び出す場合とポインタから呼び出す場合
- 関数は値を明示的に渡す必要がある
- メソッドは値でもポインタでも呼び出せる（Go が自動変換）

### 7. [receiver-choice - 値レシーバーとポインタレシーバーの選択](./receiver-choice/README.md)

値レシーバーとポインタレシーバーのどちらを選ぶべきかの判断基準を学びます。

- ポインタレシーバーを使う理由（値を変更する、効率性）
- 一貫性の重要性
- 実践的なガイドライン

## 実行方法

### 個別に実行

各ディレクトリに移動して実行：

```bash
cd struct && go run main.go
```

### 全ての例を一括実行

```bash
./runMethodBasic.sh
```

## 学習のポイント

1. **メソッドの基本**: 構造体にメソッドを定義する方法
2. **関数との違い**: メソッドと通常の関数の違い
3. **型の拡張**: 構造体以外の型にもメソッドを定義できる
4. **レシーバーの種類**: 値レシーバーとポインタレシーバーの使い分け
5. **間接参照**: 関数とメソッドの呼び出し方の違い
6. **選択基準**: 値レシーバーとポインタレシーバーのどちらを選ぶべきか
